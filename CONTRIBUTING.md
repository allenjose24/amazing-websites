# Contributing Guidelines

Thank you for your interest in contributing! We appreciate your help in making this project better.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please check the existing issues to see if it has already been reported. If not, open a new issue and include:
* A clear, descriptive title.
* Steps to reproduce the issue.
* Expected behavior vs. actual behavior.
* Relevant screenshots or logs.

### Suggesting Enhancements
We welcome ideas for new features! To propose a change:
* Open an issue or start a Discussion under the **Ideas** category.
* Describe the feature, why it would be useful, and how it could be implemented.

### Submitting Pull Requests
1. **Fork the Repository**: Create your own copy of the repository.
2. **Create a Branch**: Create a branch for your changes (e.g., \`feat/new-animation\` or \`fix/button-alignment\`).
3. **Implement Changes**: Ensure your code is clean and adheres to the styling conventions.
4. **Test Locally**: Run \`npm run dev\` and check that everything works.
5. **Open a PR**: Submit a Pull Request into the \`main\` branch. Provide a clear description of your changes and link any related issues (e.g., \`Closes #18\`).

## Local Setup

To set up the project locally:
1. Clone the repository: \`git clone https://github.com/allenjose24/amazing-websites.git\`
2. Install dependencies: \`npm install\`
3. Start the local server: \`npm run dev\`
