# GitHub Wiki Hosting Guide 🚀

This guide explains how to publish the documentation files in the `docs/` directory so they are visible on your GitHub repository.

---

## ℹ️ How GitHub Wikis Work Under the Hood

Every GitHub repository has a built-in Wiki system. Under the hood, **a GitHub Wiki is a separate Git repository** associated with your main repository. 
* If your main repository URL is: `https://github.com/username/repo-name`
* The wiki repository is located at: `https://github.com/username/repo-name.wiki.git`

This guide covers two methods for publishing the documentation:
1. **Method 1 (Recommended)**: Publishing to the official **Wiki tab** of your GitHub repository.
2. **Method 2**: Integrating the docs directly into your **main code branch** under the `docs/` folder.

---

## 🛠️ Method 1: Publishing to the Official GitHub Wiki Tab (Recommended)

Follow these steps to upload the markdown documentation files to the Wiki tab.

### Step 1: Enable Wikis in Your Repository Settings
1. Open your repository on GitHub.com.
2. Click the **Settings** (gear icon) tab in the top navigation bar.
3. In the sidebar under **General**, scroll down to the **Features** section.
4. Check the **Wikis** checkbox to enable the feature.

---

### Step 2: Initialize the Wiki Git Repository
Before you can clone the wiki via Git, you must create a placeholder page on the GitHub interface to initialize the repository:
1. Click the newly visible **Wiki** tab in the main repository navigation bar.
2. Click the green **Create the first page** button.
3. GitHub automatically creates a page named `Home`. Add a short description (e.g., "The Vault documentation hub") in the text area.
4. Scroll down and click **Save Page**.

---

### Step 3: Clone the Wiki Repository to Your Local Machine
Open a terminal on your computer and clone the wiki repository:

```bash
# Clone the wiki repository (separate from your main code repo)
git clone https://github.com/username/repo-name.wiki.git
```
> [!IMPORTANT]
> Make sure to replace `username` with your GitHub username (or organization name) and `repo-name` with your actual repository name (e.g., `https://github.com/allenjose24/amazing-websites.wiki.git`).

---

### Step 4: Copy the Markdown Files Into the Wiki Repository
Copy the files from your main repository's `docs/` folder into the newly cloned wiki folder:

1. Copy the following files:
   * `Home.md`
   * `Architecture-&-Flows.md`
   * `Database-Schema-&-RLS.md`
   * `Edge-Functions.md`
   * `Frontend-Structure-&-State.md`
   * `Interactive-UI-Components.md`
   * `GitHub-Wiki-Hosting-Guide.md`
2. Paste them into the root of the cloned `repo-name.wiki` folder.
3. Replace the default placeholder `Home.md` generated by GitHub in Step 2 with the copy from the `docs/` directory.

---

### Step 5: Configure a Custom Sidebar Navigation Menu
GitHub Wikis allow you to create a custom sidebar menu. To do this:
1. Create a new file named `_Sidebar.md` in the root of your cloned wiki repository.
2. Paste the following navigation structure into it:

```markdown
### 🏛️ The Vault Docs
* [[Home]]
* [[Architecture & Flows\|Architecture-&-Flows]]
* [[Database Schema & RLS\|Database-Schema-&-RLS]]
* [[Edge Functions\|Edge-Functions]]
* [[Frontend Structure & State\|Frontend-Structure-&-State]]
* [[Interactive UI Components\|Interactive-UI-Components]]
* [[GitHub Wiki Hosting Guide\|GitHub-Wiki-Hosting-Guide]]
```

---

### Step 6: Commit and Push to the Remote Wiki Repository
Push the files to GitHub to publish them:

```bash
# Navigate to the cloned wiki directory
cd repo-name.wiki

# Add the new documents
git add .

# Commit changes
git commit -m "Publish comprehensive technical documentation suite"

# Push to GitHub (usually master or main branch)
git push origin master
```

Once pushed, go to the **Wiki** tab of your repository on GitHub.com to view the formatted homepage and sidebar menu.

---

## 📂 Method 2: Integrating Docs Directly Into the Main Codebase

If you prefer not to manage a separate wiki repository, you can keep the `docs/` directory inside your main branch. GitHub automatically renders markdown files and resolves relative links directly within the standard file browser.

### Commit and Push to the Main Branch:
Open a terminal in your project directory and run:

```bash
# Stage the documentation folder
git add docs/

# Commit changes
git commit -m "docs: add comprehensive wiki documentation folder"

# Push code to your remote repository
git push origin main
```

### Viewing the Docs:
Users can open the `docs/` folder directly on GitHub and click `Home.md`. GitHub renders relative page references (e.g. `[Architecture](./Architecture-&-Flows.md)`) automatically, allowing users to navigate between documents.
